# Семинар: ranges и полезные возможности современного C++

Этот архив собран максимально просто:
каждый файл — отдельный пример с `main`, который можно запускать напрямую через `g++`.

## Как запускать

Общий шаблон:

```bash
g++ -std=c++23 -Wall -Wextra -Wpedantic examples/03_views_filter_transform_take.cpp -o example
./example
```

Если хочется, можно использовать и `-std=c++20` почти для всех файлов,
но для единообразия проще везде писать `-std=c++23`.

## Логика семинара

### Блок 1. Зачем вообще ranges
- Показать, что старые алгоритмы часто требовали передавать пару итераторов.
- Обсудить, что в новом стиле можно передавать контейнер целиком.
- Зафиксировать: `ranges` — это не новый контейнер, а новый способ работать с диапазонами.

### Блок 2. Views и pipeline
- Показать `filter`, `transform`, `take`.
- Затем показать `iota`, `drop`, `reverse`.
- После этого обязательно обсудить ленивость и тот факт, что view обычно не владеет данными.

### Блок 3. Когда нужен обычный контейнер
- Показать, что view — это удобно, но иногда нужен реальный `std::vector`.
- Для этого материализуем результат через `std::ranges::copy`.

### Блок 4. Еще несколько полезных modern C++ инструментов
- `std::span` — единый интерфейс для массива, `std::array`, `std::vector`
- `concepts` — явные требования к шаблонам
- `std::format` — аккуратный вывод
- `std::source_location` — удобное логирование
- `std::erase_if` — маленькое практическое улучшение в повседневном коде

## Методические акценты

### Что важно проговорить вслух
- view — это обычно не контейнер
- вычисления во view часто ленивые
- изменения исходных данных могут повлиять на view
- `span` не владеет данными
- `concepts` улучшают сообщения об ошибках и делают намерения программиста явнее

### Где студенты чаще всего путаются
- ожидают, что pipeline сразу создает новый контейнер
- думают, что view хранит копию данных
- не видят разницы между "лениво описали преобразование" и "реально посчитали"
- не понимают, зачем `span`, если уже есть указатель и длина

## Замечание про `std::format`

На современных компиляторах пример обычно собирается нормально.
Если на конкретной машине `std::format` вдруг не поддержан, этот файл можно просто пропустить:
остальные примеры от него не зависят.
